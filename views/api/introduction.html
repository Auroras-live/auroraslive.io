Introduction

The Auroras.live API allows you to obtain key aurora hunting information and access extensive data archives. It's also the system used by the Auroras.live website, Pebble watch app and mobile apps.

Usage

Timezones

How to use the API

All calls are made over HTTPS via GET or POST

Example

https://api.auroras.live/v1/

Version bumps occur when there is a breaking change. Feature additions will not cause a bump

Important Messages

Important messages are included with each request via the message property:

Example

http://api.auroras.live/v1/?type=ace&data=bz 

{
  "date": "2016-03-31T02:53:16+11:00",
  "message": [
    "Version 0 of the API is being deprecated. Please migrate to v1. Version 0 will be unpublished on January 1st 2017",
    "There will be an outage on the 8th of July 2016 between 00:00 and 01:00 UTC while services are moved to a new host"
  ],
  "bz": "1.7",
  "colour": "yellow"
}

Dates and Timezones

By default all dates and times returned are UTC. You can adjust this by adding &tz to your request, along with an offset in minutes. If you're using Javascript, you can simply call new Date().getTimezoneOffset() to retrieve this

Example

http://api.auroras.live/v1/?type=ace&data=bz&tz={{ $root.timezone }}

Returns the colour in the specified format

Formats (BETA)

By default, data will be returned as JSON. If you wish to return the data as XML, simply append format=xml to your request. Note: this won't work on all modules, as the generated fields contain illegal keys

Valid Values

Value Notes
xml Returns the colour as an XML string. May not work for all modules

Example

http://api.auroras.live/v1/?type=ace&data=bz&format=xml

Returns

Returns the colour in the specified format

See Example 

{{ $root.resultsData }}

Colours

Many modules will return a colour along with the information. This allows users to know what the returned values actually mean (e.g. green = no activity, red = lots of activity). It's also designed to allow programmers to set colours without having to code in lenghthy switch statements.

If you wish to change the format of the colours, simply append colour= to your request along with hex or rgb to return the colours in that format.

Please take note of spelling, as the API uses British English, so colour as opposed to color

Valid Values

Value Notes
hex Returns the colour as a hexadecimal string
rgb Returns the colour as a rgb triplet (CSS property)

Example

http://api.auroras.live/v1/?type=ace&data=bz&colour=hex

Returns

Returns the colour in the specified format

See Example 

{{ $root.resultsData }}
