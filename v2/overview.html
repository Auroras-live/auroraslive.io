Auroras.live v2 API Overview

Version 2 of the Auroras.live API is completely re-written from the ground up to be faster and more flexible, with new data and features not found in the previous version.

Differences between versions

  • The API is now accessed by POSTing a JSON payload to the desired endpoint
    • Though some data can still be accessed via GET and traditional POST methods
  • Data is now broken up into individual modules. For example bz is no longer grouped under the ace module
  • Timezone offset can now be either a number (e.g. -600) or a timezone name (e.g. Australia/Melbourne)

Making an API call

To make an API call, send a POST request with a JSON body to {{ site.api_url }}/v2. This JSON body should have a key that corresponds to the module you wish to call, and the value of the key should be the parameters for that module:

  {
    "weather": {
      "lat": -12,
      "long": 34,
    }
  }

About Dot Notation

This documentation makes use of "dot notation", which, unless stated otherwise, denotes a property of the JSON object. For example alerts.children would be equal to:

  {
    "alerts": {
      "children": ...
    }
  }

This also applies to GET requests, so the same alerts.children would be equal to {{ site.api_url }}/v2/alerts?alerts[children]=...
